# ドキュメント品質レビュー

**目的:** `docs/core/`、`CLAUDE.md`、`README.md`のドキュメント品質をレビューし、改善提案を行います。

**引数:**
- `/review-docs` - 全ドキュメントをレビュー（CLAUDE.md、README.md含む）
- `/review-docs [ファイル名]` - 指定したドキュメントをレビュー（例: `architecture.md`）
- `/review-docs [モジュール名]` - 指定したモジュールに関連するドキュメントのみをレビュー（例: `jquants`）
- `/review-docs [steeringフォルダ名]` - `.steering/` 内のフォルダを指定し、その機能に関連するドキュメントのみをレビュー（例: `20260207-slack-notification`）
- `/review-docs --from-report [レポートパス]` - 前回のレビューレポートの指摘事項が修正されたかを中心に再レビュー

**`--from-report`の引数解釈:**

| 入力 | 解釈 |
|------|------|
| `--from-report 20260215-x-trend-v2` | `.steering/20260215-x-trend-v2/` 内の最新の `review-report*.md` を対象 |
| `--from-report .steering/20260215-x-trend-v2/review-report.md` | 指定ファイルのみ |

**モード判定:**
1. 引数に `--from-report` が含まれる場合 → **再レビューモード**
2. 引数が `YYYYMMDD-` 形式で始まる場合 → steeringフォルダ指定
3. それ以外 → 従来のファイル名/モジュール名判定

**フォルダ名 vs その他の判定（通常モード）:**
- `YYYYMMDD-` 形式（例: `20260207-slack-notification`）で始まる場合 → steeringフォルダ指定
- それ以外 → 従来のファイル名/モジュール名判定

**出力:**
- `--from-report`指定時: 元レポートと同じフォルダに `review-report-N.md`（連番付与）で保存
- steeringフォルダ指定時: `.steering/[フォルダ名]/review-report.md` に保存（既存ファイルがある場合は `review-report-N.md` で連番付与）
- それ以外: `.steering/YYYYMMDD-docs-review/review-report.md` に保存（同名フォルダが存在する場合は連番を付与）

## レビュー観点

| # | 観点 | 内容 |
|---|------|------|
| 1 | 完全性 | 全モジュール・APIが文書化されているか |
| 2 | 最新性 | コードの変更がドキュメントに反映されているか |
| 3 | 正確性 | 記述が実際のコードと一致しているか |
| 4 | 分かりやすさ | 読みやすく理解しやすいか |
| 5 | 一貫性 | 命名・フォーマットが統一されているか |


## レビューの流れ（8ステップ）

**各ステップの詳細手順・スコープ判定ロジック・出力フォーマットは [reference.md](./reference.md) を参照してください。レビュー実行前に必ず reference.md を読むこと。**

| ステップ | 内容 |
|----------|------|
| 1 | レビュー対象の特定（引数判定／steeringフォルダ・モジュールのスコープ決定） |
| 2 | 完全性チェック（モジュール/APIカバレッジ） |
| 3 | 最新性チェック（git log でコードとの更新差分） |
| 4 | 正確性チェック（シグネチャ・パス・CLAUDE.md/README.md の整合） |
| 5 | 分かりやすさチェック（説明充実度・曖昧表現検出） |
| 6 | 一貫性チェック（命名規則・フォーマット・用語統一） |
| 7 | レビューレポートの生成 |
| 8 | レポートの保存（`.steering/` へ、既存時は連番付与） |

## 評価基準

| 評価 | 基準 |
|------|------|
| **A** | 問題なし |
| **B** | 軽微な問題1-3件 |
| **C** | 問題4-10件 |
| **D** | 問題10件以上または重大な問題あり |

### 総合評価

- **A**: 全観点がA
- **B**: 最低評価がB
- **C**: 最低評価がC
- **D**: 1つでもDがある

## 再レビューモード（--from-report）

`--from-report` 指定時は通常レビューに代えて、前回レポートの指摘事項の修正状況を中心に再レビューします（F1: レポート読込 → F2: 指摘抽出 → F3: 修正状況チェック → F4: 新規問題検出 → F5: 再レビューレポート生成）。手順と出力フォーマットは [reference.md](./reference.md) を参照してください。

## 注意事項

- レビューは読み取りのみで、ドキュメントを変更しません
- レビューレポートは `.steering/` に自動保存されます
- 自動検出には限界があるため、人間による確認も推奨
- 問題の修正は `/update-docs` または手動で行ってください
